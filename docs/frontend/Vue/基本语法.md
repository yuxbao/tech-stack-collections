vue是一个响应式框架，用于构建动态渲染的html页面，其采用组件化方式构建项目。

### 安装Vue项目

:::code-tabs
@tab pnpm
```sh
pnpm create vue@latest
```
:::

使用vue-create-app+vite进行构建初始化vue项目。


### 基本概念

#### app应用实例

每个vue应用都有一个app或者多个app，在main.js中通过`createApp`函数创建新的**应用实例**。

传入的参数作为应用的**根组件**

```js
import { createApp } from 'vue'
// 从一个单文件组件中导入根组件
import App from './App.vue'

const app = createApp(App)  // 这里App.vue作为根组件
```

#### 挂载

应用实例需要挂载才能被渲染，需要选择一个实际的DOM元素。

```html
<div id="app"></div>
```

```js
app.mount('#app')
```

容器元素自己不会被认为是应用的一部分，所以`app`不是应用一部分，而根组件`div`是应用的一部分。

`.mount()`方法在所有资源配置注册完成后才被调用，返回一个根组件而不是应用实例。

### 模板语法

#### 插值

Mustache双大括号语法：
```html
<span>Message: {{ msg }}</span>
```

内容被替换为对应组件的实例属性值。

支持js表达式：

```js
{{ number + 1 }}

{{ ok ? 'YES' : 'NO' }}

{{ message.split('').reverse().join('') }}

<div :id="`list-${id}`"></div>
```

#### 属性绑定

`v-bind`语法响应式绑定一个attribute。也可以简化为`:[attibute-name]`这样子，用来绑定html元素的内部属性控制，或者传递props给子元素。

#### 指令Directives

指令`v-[xxx]`是特殊的attributes，利用Vue提供的内置指令实现。包括`v-bind`，`v-html`等。

>[!note]
>一个指令的任务是在其表达式的值变化后响应式的更新DOM。

```html
<p v-if="seen">Now you see me</p>
```

### 响应式

#### ref

==组合式API(Composition API)==中，推荐使用`ref()`函数声明响应式状态。

```js
import { ref } from 'vue'

const count = ref(0)
```

ref会接受参数，将其包装为带有`.value`属性的对象中。

```js
const count = ref(0)

console.log(count) // { value: 0 }
console.log(count.value) // 0

count.value++
console.log(count.value) // 1
```

>在模板语法中使用ref时，不需要加`.value`，vue会自动解包。但是在js中还是要使用的。

因为使用了`<script setup>`语法糖，所以不需要手动暴露状态和方法。

ref会自动追踪深层响应式。

`DOM更新时机`: 当你修改了响应式状态时，Vue会周期性缓冲所有状态修改然后异步更新。


#### reactive

`reactive`把对象或者数组包装为一个响应式的对象，也是深层响应式。

缺点：
- reactvie只能追踪对象或者数组，不能包装原始对象。
- 解包属性时会出现响应式链接丢失情况。

所以不推荐使用，还是推荐`ref`。
