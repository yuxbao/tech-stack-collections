vue 是一个响应式框架，用于构建动态渲染的 html 页面，其采用组件化方式构建项目。

### 安装 Vue 项目

:::code-tabs
@tab pnpm

```sh
pnpm create vue@latest
```

:::

使用 vue-create-app+vite 进行构建初始化 vue 项目。

### 基本概念

#### app 应用实例

每个 vue 应用都有一个 app 或者多个 app，在 main.js 中通过`createApp`函数创建新的**应用实例**。

传入的参数作为应用的**根组件**

```js
import { createApp } from "vue";
// 从一个单文件组件中导入根组件
import App from "./App.vue";

const app = createApp(App); // 这里App.vue作为根组件
```

#### 挂载

应用实例需要挂载才能被渲染，需要选择一个实际的 DOM 元素。

```html
<div id="app"></div>
```

```js
app.mount("#app");
```

容器元素自己不会被认为是应用的一部分，所以`app`不是应用一部分，而根组件`div`是应用的一部分。

`.mount()`方法在所有资源配置注册完成后才被调用，返回一个根组件而不是应用实例。

### 模板语法

#### 插值

Mustache 双大括号语法：

```html
<span>Message: {{ msg }}</span>
```

内容被替换为对应组件的实例属性值。

支持 js 表达式：

```js
{{ number + 1 }}

{{ ok ? 'YES' : 'NO' }}

{{ message.split('').reverse().join('') }}

<div :id="`list-${id}`"></div>
```

#### 属性绑定

`v-bind`语法响应式绑定一个 attribute。也可以简化为`:[attibute-name]`这样子，用来绑定 html 元素的内部属性控制，或者传递 props 给子元素。

#### 指令 Directives

指令`v-[xxx]`是特殊的 attributes，利用 Vue 提供的内置指令实现。包括`v-bind`，`v-html`等。

> [!note]
> 一个指令的任务是在其表达式的值变化后响应式的更新 DOM。

```html
<p v-if="seen">Now you see me</p>
```

### 响应式

#### ref

==组合式 API(Composition API)==中，推荐使用`ref()`函数声明响应式状态。

```js
import { ref } from "vue";

const count = ref(0);
```

ref 会接受参数，将其包装为带有`.value`属性的对象中。

```js
const count = ref(0);

console.log(count); // { value: 0 }
console.log(count.value); // 0

count.value++;
console.log(count.value); // 1
```

> 在模板语法中使用 ref 时，不需要加`.value`，vue 会自动解包。但是在 js 中还是要使用的，这就是 ref 的自动解包机制。
> 此外，在模板渲染中只有顶层 ref 对象属性才会被自动解包

因为使用了`<script setup>`语法糖，所以不需要手动暴露状态和方法。

ref 会自动追踪深层响应式。

`DOM更新时机`: 当你修改了响应式状态时，Vue 会周期性缓冲所有状态修改然后异步更新。

#### reactive

`reactive`把对象或者数组包装为一个响应式的对象，也是深层响应式。

缺点：

- reactvie 只能追踪对象或者数组，不能包装原始对象。
- 解包属性时会出现响应式链接丢失情况。

所以不推荐使用，还是推荐`ref`。

### 计算属性

推荐使用计算属性来描述依赖响应式状态的复杂逻辑，其实就是==封装一套模板语法在函数里==。

::: demo vue

```vue
<!--  vue 代码 -->
<script setup>
  import { reactive, computed } from "vue";

  const author = reactive({
    name: "John Doe",
    books: [
      "Vue 2 - Advanced Guide",
      "Vue 3 - Basic Guide",
      "Vue 4 - The Mystery",
    ],
  });

  // 一个计算属性 ref
  const publishedBooksMessage = computed(() => {
    return author.books.length > 0 ? "Yes" : "No";
  });
</script>

<template>
  <p>Has published books:</p>
  <span>{{ publishedBooksMessage }}</span>
</template>
```

:::

`computed()`方法期望接受[getter](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Functions/get#description)函数，计算属性也会在模板语法自动解包。

计算属性是基于它们的依赖进行缓存的，只有在相关响应式状态发生变化时才会重新计算。

### Class 与 Style 绑定

vue 提供了 class 与 style 样式的属性绑定增强。

```vue
<div :class="{ active: isActive }"></div>

<div class="static" :class="{ active: isActive, 'text-danger': hasError }">
</div>

<div :class="[activeClass, errorClass]"></div>
```

当`isActive`为 true 时，`div`会有`active`类。

也可以使用返回对象的计算属性监听 class，或者传递数组来渲染多个 class。

### 条件渲染

#### v-if

`v-if` 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回==真值==时才被渲染。

这是惰性渲染的。

```vue
<h1 v-if="awesome">Vue is awesome!</h1>
```

#### v-else

同时配合`v-if`使用。

:::demo vue

```vue
<template>
  <button @click="awesome = !awesome">Toggle</button>

  <h1 v-if="awesome">Vue is awesome!</h1>
  <h1 v-else>Oh no 😢</h1>
</template>
<script setup>
  import { ref } from "vue";

  const awesome = ref(true);
</script>
```

:::

#### v-else-if

相当于`else if`语句。

```html
<div v-if="type === 'A'">A</div>
<div v-else-if="type === 'B'">B</div>
<div v-else-if="type === 'C'">C</div>
<div v-else>Not A/B/C</div>
```

使用`<template>`标签包裹多个元素，在其上添加`v-if`就可以切换多个元素。

#### v-show

用法和`v-if`类似，但是`v-show`只是简单地切换元素的 CSS `display`属性，在 DOM 上保留元素，不支持在 template 上使用。

这不是惰性渲染的。
